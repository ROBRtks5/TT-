# ПРОТОКОЛ ОМНИ-АУДИТА (TITAN OMNI-AUDIT)

**Цель:** Тотальная проверка 100% кодовой базы на скрытые баги, паразитные механики, математические неточности, утечки памяти, и мертвый код.

## 🟩 ФАЗА 1: Ядро и Жизненный Цикл (Boot & Lifecycle)
- [x] 1.1. Аудит `SafeBootGuard` и `ErrorBoundary` (удаление `window.location.reload()`, мягкие методы восстановления).
- [x] 1.2. Проверка хуков `useEffect` во всех глобальных провайдерах на утечки при монтировании/размонтировании (cleanup functions).
- [x] 1.3. Анализ `main.tsx` и `App.tsx` на предмет "узких мест" рендеринга (bottlenecks).
- [x] 1.4. Проверка хранилища (IndexDB/LocalStorage) на предмет переполнения (quota limits).

## 🔲 ФАЗА 2: Мост Ядра (Worker IPC & Memory)
- [x] 2.1. Аудит `useTradingBot.ts` и `bot-kernel.ts` на предмет гонок (Race Conditions) при `postMessage`.
- [x] 2.2. Проверка сериализации State. Нет ли утечек памяти из-за бесконечно растущих массивов (Trade History, Logs).
- [ ] 2.3. Зачистка мертвого кода, оставшегося после старой архитектуры скрейпинга интерфейса.

## 🔲 ФАЗА 3: Сетевая Броня (Networking & Streams)
- [x] 3.1. `streamManager.ts` & `tradeStreamService.ts`: Умная обработка 429 (Too Many Requests), 500, 503 ошибок биржи.
- [x] 3.2. `tInvestService.ts`: Полный аудит вызовов gRPC/REST. Проверка таймаутов, отсутствие "мертвых зависаний" (hanging requests).
- [x] 3.3. Разрыв стримов: тестирование восстановления без потери State.

## 🔲 ФАЗА 4: Торговая Логика и Математика (Math Precision & State)
- [x] 4.1. `mathStrategyService.ts`: Проверка расчетов сетки, ATR, Bollinger. Избежание ошибок точности `float` (например, 0.1 + 0.2 = 0.3000000000004).
- [x] 4.2. `capitalService.ts`: Строгая проверка распределения 40/40/20. Защита от превышения маржи и "Denial of Wallet".
- [x] 4.3. Округление лотов, цен и шага цены (minPriceIncrement, lotSize validation).

## 🔲 ФАЗА 5: Исполнение и Контроль (Order Controller)
- [x] 5.1. `OrderController.ts`: Логика обработки частичных исполнений (Partial Fills).
- [x] 5.2. Отмененные биржей ордера: автоматическое восстановление логики сетки, чтобы не было "дыр".
- [x] 5.3. Защита от спама ордерами (Rate Limiting на создание заявок).
