# 📱 TITAN PROTOCOL: ANDROID MOBILE ARCHITECTURE

## 1. ВВЕДЕНИЕ
Настоящий документ описывает текущую мобильную архитектуру TITAN-а. В соответствии с директивой перехода на Native Android, в этот репозиторий были интегрированы все необходимые мосты, плагины и системные настройки для того, чтобы приложение не просто открывалось в WebView, но и функционировало как полноценный фоновый процесс. 

## 2. СДЕЛАННЫЕ ШАГИ И ВНЕДРЕННЫЙ ФУНКЦИОНАЛ

### 2.1 Генерация файловой системы Android (Capacitor)
Проект был трансформирован в Capacitor-приложение. Был запущен `npx cap add android`, который сгенерировал полноценную папку `android/`.
**Что это значит для вас:** Вам НЕ нужно пересобирать проект с нуля. При загрузке архива в Android Studio (папка `android`), среда разработки сама подтянет Gradle и обернет наш React/Vite-бот во внутреннюю Native-оболочку.

### 2.2 Интеграция Android-моста (`androidBridge.ts`)
Был написан специализированный сервис `src/services/androidBridge.ts`. Он делает следующее:
1. Автоматически понимает, работает TITAN в браузере или на Android-устройстве (`Capacitor.isNativePlatform()`).
2. Опрашивает операционную систему на изменение состояния приложения (`appStateChange`).
3. Принудительно захватывает WakeLock экрана.

### 2.3 Плагины и Зависимости 
Был установлен ряд плагинов:
- `@capacitor/core` и `@capacitor/app`: Ядро и доступ к состоянию Activity.
- `@capacitor-community/keep-awake`: Плагин для работы с дисплеем и запрета сна.
- `@capacitor/haptics`: Для тактильной отдачи (фидбека) при нажатиях.

### 2.4 Настройка AndroidManifest.xml (КРИТИЧЕСКИ ВАЖНО)
Вручную изменен `android/app/src/main/AndroidManifest.xml`. Добавлены следующие пермиссии (разрешения), которые будут запрошены у пользователя при установке:
- `android.permission.INTERNET` — доступ к серверам Tinkoff API.
- `android.permission.WAKE_LOCK` — доступ к энергопотреблению (CPU/экран) для запрета сна.
- `android.permission.REQUEST_IGNORE_BATTERY_OPTIMIZATIONS` — **Ключевой флаг!** Отключает экономный режим Android (Doze Mode). Благодаря ему система не убьет бота (процесс), даже если вы свернете приложение или выключите экран.
- `android.permission.FOREGROUND_SERVICE` — Подготовка к работе 24/7 в статус-баре. Если понадобится держать бота активным неделями, это разрешение обеспечит легальное существование сервиса в памяти.

## 3. ИСТОРИЯ ПРОБЛЕМ И РЕШЕНИЙ ВО ВРЕМЯ ИНТЕГРАЦИИ
*(Для последующих ИИ)*

**🚨 ПРОБЛЕМА 1: React Crash -> "Invalid hook call: Cannot read properties of null (reading 'useState')"**
* **Симптомы:** После первичного подключения `initializeAndroidBridge()` в корень `index.tsx`, приложение упало с Critical Error: Invalid hook call внутри компонента `SafeBootGuard`.
* **Анализ:** Ошибка "Invalid Hook Call" обычно возникает при конфликте версий React или если внешняя не-React библиотека вмешивается в порядок инициализации среды (DOM/Node env) _до_ монтирования `createRoot`. Так как Capacitor-плагины (`@capacitor/app`) лезут в глобальные объекты `window` и пытаются инициализировать Native мосты, вызов их на глобальном уровне `index.tsx` до `React.createElement` разрушал контекст хуков.
* **Решение:** Вызов `initializeAndroidBridge();` был изъят из глобальной области `index.tsx` и помещен строго внутрь `useEffect` корневого компонента `App.tsx`! Таким образом, мост инициализируется только после того, как React успешно смонтировался. Ошибка исчезла.

**🚨 ПРОБЛЕМА 2: Peer Dependencies Collision Capacitor v6 vs React 18 / 19**
* **Симптомы:** При установке плагинов `keep-awake` и `background-task` NPM выдавал фатальные конфликты 404 и ERESOLVE.
* **Решение:** Проведен строгий менеджмент версий: `@capacitor/android@^6.0.0` и `@capacitor-community/keep-awake@^5.0.0`. Версии жестко привязаны друг к другу для Capacitor 6. `background-task` исключен в пользу нативных флагов Manifest и хуков AppState.

## 4. ДАЛЬНЕЙШИЕ ИНСТРУКЦИИ ДЛЯ ПОЛЬЗОВАТЕЛЯ
Что делать дальше:
1. Скачать этот проект со всеми папками.
2. Открыть **Android Studio**.
3. Выбрать **Open Project** -> выбрать папку `android` внутри скачанного проекта! 
4. Дождаться загрузки Gradle (может занять 5-10 минут).
5. Нажать кнопку Run (зеленый треугольник) для установки на ваш телефон из Android Studio.
6. В самом телефоне, после установки, ОБЯЗАТЕЛЬНО зайдите в настройки приложения -> Батарея -> Выберите "Не ограничено" (Unrestricted) / Отключите оптимизацию.

Бот готов. Мобильное Ядро установлено.
