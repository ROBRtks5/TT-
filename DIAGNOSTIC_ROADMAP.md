# TITAN Protocol Diagnostic Roadmap

Используем кодовое слово `TITAN_CHECK` для итеративной проверки.

---

## План диагностики (Checklist)

### Фаза 1: Сетевой уровень и Аутентификация
- [ ] 1.1 Проверка статуса авторизации и токена (tInvestService).
- [ ] 1.2 Проверка соединения с брокером (WebSocket/gRPC).

### Фаза 2: Поток и целостность данных
- [ ] 2.1 Проверка актуальности данных (последняя свеча vs реальное время).
- [ ] 2.2 Проверка получения тиковой информации (lastTrades).

### Фаза 3: Логика формирования стратегии (AMM)
- [ ] 3.1 Вывод входных параметров расчета сетки (текущая цена, полосы Боллинджера, волатильность).
- [ ] 3.2 Логирование результата функции `calculateAMMGrid` (почему сетка пуста?).

### Фаза 4: Управление и исполнение ордеров
- [ ] 4.1 Проверка доступного BuyingPower (почему ноль?).
- [ ] 4.2 Проверка попыток вызова `executeOrder` (логи ошибок брокера).

### Фаза 5: Реконсиляция
- [ ] 5.1 Сверка локального состояния (activeGridOrders) с реальными данными на бирже.
