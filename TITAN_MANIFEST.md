
# ПРОТОКОЛ "ТИТАН": МАНИФЕСТ РАЗРАБОТЧИКА (РЕДАКЦИЯ 43.3.0 - GOLD MASTER)

> **ВНИМАНИЕ:** Код очищен от "маркетинговой шелухи". Осталась только боевая логика.

---

## 1. АРХИТЕКТУРНЫЕ ПРИНЦИПЫ

### I. ПРИОРИТЕТ МАТЕМАТИКИ (MATH OVER AI)
*   Мы больше не полагаемся на "настроение" ИИ.
*   Решение принимает только `mathStrategyService`.
*   Формула проста: `Цена Входа` -> `Шаг Сетки` -> `Лимитный Ордер`.

### II. ГЛУБОКОЕ СКАНИРОВАНИЕ (DEEP SCAN)
*   **Авто-режим:** При старте бот скачивает историю за 180 дней.
*   **Принцип 5%:** Мы находим уровень цены, ниже которого рынок был только 5% времени (исключая флеш-крэши).
*   **Распределение:** Сетка пуль (2, 3 или 4) растягивается так, чтобы последний выстрел приходился на этот исторический минимум.

### III. БЕЗОПАСНОСТЬ ИСПОЛНЕНИЯ
*   **Zero-Leverage:** Торговля только на свои.
*   **Schrodinger Protocol:** Если ответ от API не получен, ордер не исчезает, а переходит в режим "Ожидания подтверждения" (30 сек).
*   **Smart Watchdog:** Интеллектуальная проверка соединения. Спит ночью, бдит днем.

### IV. ДИНАМИКА ИСПОЛНЕНИЯ (FUZZY LOGIC)
*   **Sticky Orders:** Бот не должен быть перфекционистом.
*   Если в стакане уже стоит наш ордер на продажу, и он находится в зоне профита, **мы его не трогаем**, даже если алгоритм говорит, что идеальная цена отличается на 0.01.
*   Постоянное переставление ордеров (Thrashing) приводит к потере приоритета в стакане и спаму API.
*   **Tolerance:** Допуск на отклонение цены составляет 0.5%.

*   **Ядро:** TypeScript + Web Worker (логика).
*   **UI:** React + Tailwind (визуализация).
*   **Данные:** T-Invest OpenAPI (стриминг).
*   **Storage:** IndexedDB (холодное хранение настроек).
