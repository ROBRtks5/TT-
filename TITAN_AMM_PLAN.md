# 🧬 TITAN AMM: ПЛАН ЭВОЛЮЦИИ (ГИБРИДНАЯ СТРАТЕГИЯ)

Этот документ является интерактивным планом реализации гибридной торговой системы (AMM + Ratchet) с использованием серверных мощностей Tinkoff API v2.
AI-агент использует этот файл для отслеживания прогресса.

## 📡 ФАЗА 1: Модернизация API (Перенос расчетов на сервер)
- [x] **Шаг 1.1: Интеграция GetTechAnalysis.** Добавить в `tInvestApi.ts` и `tInvestService.ts` метод для запроса линий Боллинджера (Bollinger Bands) с серверов брокера.
- [x] **Шаг 1.2: Подготовка TradesStream.** Заложить архитектуру для WebSocket-стрима исполнения ордеров (или агрессивного пуллинга конкретных ID), чтобы моментально реагировать на исполнение лимиток.

## 💰 ФАЗА 2: Реструктуризация Капитала (Правило 40/40/20)
- [x] **Шаг 2.1: Обновление типов.** Внедрить в `types.ts` разделение позиций на `GridAsset` (для сетки) и `CoreAsset` (для туземуна).
- [x] **Шаг 2.2: Логика распределения.** Переписать `capitalService.ts` для расчета лимитов: 40% кэша на закупку вниз, 40% акций на продажу вверх, 20% акций в неприкосновенном Ядре.

## 🧮 ФАЗА 3: Математическое Ядро (AMM Engine)
- [x] **Шаг 3.1: Расчет сетки (calculateAMMGrid).** В `mathStrategyService.ts` написать функцию, которая берет ширину канала Боллинджера и строит асимметричную сетку ордеров (прогрессивные объемы к краям канала).

## ⚙️ ФАЗА 4: Менеджер Сетки (Grid Controller)
- [x] **Шаг 4.1: Мульти-ордерность.** Обновить `OrderController.ts`, чтобы бот мог выставлять и контролировать каскад из 5-10 лимитных ордеров одновременно.
- [x] **Шаг 4.2: Рекурсивное дыхание.** Внедрить логику переворота: при исполнении ордера BUY немедленно выставлять SELL на один шаг сетки выше (и наоборот).

## 🚀 ФАЗА 5: Изоляция Ядра и Ratchet
- [x] **Шаг 5.1: Защита Ядра.** Гарантировать, что 20% CoreAsset никогда не участвуют в распродаже внутри сетки.
- [x] **Шаг 5.2: Триггер пробоя.** Настроить активацию алгоритма Ratchet (трейлинг-тейк-профит) для Ядра ТОЛЬКО в случае, если цена пробивает верхнюю линию Боллинджера.

## 🌊 ФАЗА 6: ACTIVE MARKET MAKER (AMM v4) - "Постоянное Присутствие"
- [x] **Шаг 6.1: Центрированная волатильность.** Рефакторинг `calculateAMMGrid`. Сетка больше не блокируется границами Боллинджера, а строится симметрично/центрированно относительно текущей цены. Ширина Боллинджера используется только для определения размаха «лестницы».
- [x] **Шаг 6.2: Очистка Тикеров.** Полное удаление Legacy-упоминаний тикера TMOS. Утвержден единственный целевой актив — **T (Т-Технологии)**.
- [x] **Шаг 6.3: Непрерывная подача ликвидности.** Бот переведен из режима «Ожидания входа в канал» в режим «Провайдера ликвидности». Торговля ведется постоянно, пока есть кэш или актив.
